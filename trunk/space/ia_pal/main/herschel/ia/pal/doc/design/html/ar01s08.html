8. Queries

The PAL provides a query mechanism. To enable users to perform optimal queries, 3 types of queries were introduced:

  • AttribQueries that query only the attributes of Products

  • MetaQueries that query only the attributes and meta-data of Products

  • FullQueries that are full data-mining operations

In other words, linking to Section 7.3, “Data, metadata and descriptors” of previous chapter:

  • Querying for data is done through a FullQuery
  • Querying for mandatory metadata (attributes) is done through an AttribQuery
  • Querying for general metadata is done through a MetaQuery
  • Querying for descriptors is not yet supported, but it's foreseen to develop a DescriptorQuery for this

With these types of queries, the user can optimize his search strategy, and the pool implementation can optimize the query execution.

Having 3 (or 4) different query types makes the user API cumbersome.

For alleviating this, and for simplifying the condition part of the queries, a new enhanced Query has been introduced.
