4. What is the PAL?

The Product Access Layer (PAL) provides a implementation-independent interface for astronomers, calibration scientists and developers alike to read, write and save products to persistent storage locations.

The persistent storage locations (ProductPools) can be file-based, database-based or network-based. A set of standard pool implementations are available as part of the herschel.pal package, and during the process of development will be extended. But these pools are not the only ones that need be available. Of course, any developer who wishes to write his/her own pool can do so - they need only implement the ProductPool API and follow the rules listed in Section 7.3, “What can or cannot be added to a context - Rules”.
